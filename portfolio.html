FNDE · Fundo Nacional do Desenvolvimento da Educação · G4F

Auditoria de notas fiscais com PySpark, Databricks, Genie e Power BI

No FNDE, estruturei tabelas e consultas em PySpark dentro de notebooks no Databricks para investigar erros em transações bancárias que impactavam a análise de notas fiscais. Depois, usei o Genie para acelerar a passagem desse insumo analítico para um dashboard no Power BI, com atualização automatizada e apoio direto à gestão e à auditoria.

  • Desafio: transformar dados de big data em uma visão auditável, com muitas regras, filtros e comparações entre transações e notas fiscais.
  • Estrutura da solução: o dashboard foi organizado em várias abas, cada uma explicando uma inconsistência diferente e o quanto ela afetava a análise.
  • Stack: PySpark, notebooks Databricks, modelagem de tabelas analíticas, Genie e Power BI.
  • Ponto importante: não era um papel de BI puro, mas foi necessário levar o dado estruturado até a camada de consumo para apoiar a gestão.
  • Automação: o Genie também ajudou na criação do job para atualizar e publicar o dashboard automaticamente.
  • Gancho de entrevista: engenharia analítica em big data, qualidade de dados, auditoria e integração entre camada técnica e visualização executiva.
FNDE · case complementar

Auditoria de transações bancárias com Random Forest

Projeto de machine learning criado como exemplo técnico muito próximo do tipo de problema que enfrentei no FNDE: priorização de inconsistências e sinais de risco em dados transacionais para apoiar auditoria, análise e tomada de decisão.

  • Objetivo: classificar transações suspeitas em um cenário de auditoria bancária usando uma base pública sintética, preservando a lógica do problema sem expor dados reais.
  • Pipeline: leitura em `Parquet`, parsing temporal, criação de atributos derivados, preprocessamento com `ColumnTransformer`, imputação, `OneHotEncoder` e treino com `RandomForestClassifier`.
  • Features: valor da transação, tipo, local, dispositivo, faixa horária, transação internacional e atributos derivados como `day_of_week`, `month`, `sender_prefix` e `receiver_prefix`.
  • Métricas: `F1-score = 0.9620`, `ROC-AUC = 0.9996` e `PR-AUC = 0.9967` no split atual da base sintética.
  • Interpretação: uso de `feature importance` para ranquear sinais relevantes de auditoria, como valor, tipo de transação, local e dispositivo.
  • Conexão com o FNDE: o projeto ajuda a explicar como estruturo raciocínio analítico para auditoria de transações, conciliação, priorização de inconsistências e apoio a investigações mais profundas.
  • Stack: Python, pandas, scikit-learn, matplotlib, pyarrow, joblib e unittest.
  • Gancho de entrevista: classificação supervisionada em base tabular, tratamento de base desbalanceada, explicabilidade com feature importance e adaptação de case sintético para cenário real.

Presidência da República

Baseline de regressão linear para óbitos por Covid-19 no Brasil

Projeto desenvolvido a partir de dados abertos oficiais para modelar a tendência de óbitos por Covid-19 no Brasil com um baseline clássico, interpretável e reproduzível. A proposta foi transformar microdados governamentais em uma série temporal diária e avaliar a capacidade preditiva de um modelo linear com engenharia de atributos temporais.

  • Contexto: experiência alinhada ao meu estágio na Presidência, conectando análise de dados públicos, temas de saúde e modelagem quantitativa aplicada.
  • Fonte de dados: microdados oficiais do SIVEP-Gripe / SRAG, publicados no Portal de Dados Abertos do SUS.
  • Pipeline: filtragem de casos de Covid-19 com evolução para óbito, agregação nacional por dia, reindexação completa do calendário de 2024 e construção da série alvo com média móvel de 7 dias.
  • Features: lags de curto prazo, médias móveis, desvio padrão móvel, tendência recente e variáveis de calendário como dia da semana, semana do ano e mês.
  • Modelagem: `LinearRegression` com divisão temporal treino-teste, preservando a ordem cronológica da série.
  • Métricas: `R² = 0.9420`, `MAE = 0.4576` e `RMSE = 0.5639`, mostrando um baseline forte para tendência epidemiológica suavizada.
  • Stack: Python, pandas, numpy, scikit-learn, matplotlib, joblib, unittest e reconstrução da base a partir da fonte oficial.
  • Gancho de entrevista: como transformar microdados públicos em série temporal utilizável, quando usar regressão linear como baseline e como avaliar modelos simples com boa interpretabilidade.

Search Performance Assistant para avaliação de retrieval

Desenvolvimento de uma aplicação em Python para estudar e demonstrar recuperação de documentos com TF-IDF, indexação vetorial, fallback sem FAISS e uma camada de assistant baseada em evidências recuperadas.

  • Problema: avaliar retrieval de forma prática e explicável, mostrando como uma consulta recupera os documentos certos e como medir essa qualidade.
  • Arquitetura: `SearchAssistant` para ingestão, retrieval e resposta; `TfidfEncoder` para vetorização; `SearchIndex` para indexação vetorial; `RecallEvaluator` para avaliação e `app.py` para demonstração visual.
  • Stack: Python, scikit-learn, TF-IDF, FAISS, similaridade de cosseno, Tkinter e unittest.
  • Diferencial técnico: uso de FAISS com normalização L2 quando a biblioteca está disponível e fallback com cosine similarity quando não está, mantendo a aplicação portátil.
  • Métricas: Semantic Recall@K, ID Recall@K e Hybrid Recall@K para comparar recuperação por similaridade textual, por IDs esperados e por visão combinada.
  • Caso de teste: o pipeline automatizado valida recuperação dos documentos `D2` e `D3` com `semantic_recall = 1.0`, `id_recall = 1.0` e `hybrid_recall = 1.0`.
  • Perguntas de entrevista que esse projeto sustenta: quando usar TF-IDF antes de embeddings densos, como desenhar fallback para dependência opcional, como avaliar retrieval e como explicar ranking de busca de forma rastreável.
