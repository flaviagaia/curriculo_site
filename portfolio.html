Projeto próprio

Search Performance Assistant para avaliação de retrieval

Desenvolvimento de uma aplicação em Python para estudar e demonstrar recuperação de documentos com TF-IDF, indexação vetorial, fallback sem FAISS e uma camada de assistant baseada em evidências recuperadas.

  • Problema: avaliar retrieval de forma prática e explicável, mostrando como uma consulta recupera os documentos certos e como medir essa qualidade.
  • Arquitetura: `SearchAssistant` para ingestão, retrieval e resposta; `TfidfEncoder` para vetorização; `SearchIndex` para indexação vetorial; `RecallEvaluator` para avaliação e `app.py` para demonstração visual.
  • Stack: Python, scikit-learn, TF-IDF, FAISS, similaridade de cosseno, Tkinter e unittest.
  • Diferencial técnico: uso de FAISS com normalização L2 quando a biblioteca está disponível e fallback com cosine similarity quando não está, mantendo a aplicação portátil.
  • Métricas: Semantic Recall@K, ID Recall@K e Hybrid Recall@K para comparar recuperação por similaridade textual, por IDs esperados e por visão combinada.
  • Caso de teste: o pipeline automatizado valida recuperação dos documentos `D2` e `D3` com `semantic_recall = 1.0`, `id_recall = 1.0` e `hybrid_recall = 1.0`.
  • Perguntas de entrevista que esse projeto sustenta: quando usar TF-IDF antes de embeddings densos, como desenhar fallback para dependência opcional, como avaliar retrieval e como explicar ranking de busca de forma rastreável.

Release Notes Generation Assistant com retrieval e avaliação automática

Desenvolvimento de uma aplicação em Python para geração assistida de release notes a partir de contexto de release e pull requests, combinando recuperação por similaridade, heurísticas temáticas, geração baseada em regras e métricas de avaliação.

  • Problema: transformar contexto de release e PRs em notas curtas, coerentes e rastreáveis, com critérios objetivos para validar a saída.
  • Arquitetura: pipeline modular com carregamento e validação de JSON, vetorização TF-IDF, retriever com cosine similarity e topic boosting, geração textual e camada de avaliação.
  • Stack: Python, scikit-learn, TF-IDF, similaridade de cosseno, Tkinter, JSON e unittest.
  • Diferencial técnico: uso de score híbrido no retriever, somando similaridade vetorial com reforço por grupos temáticos como acesso, busca e estabilidade.
  • Métricas: Recall@K para verificar recuperação dos PRs esperados e Exact Match para comparar a frase final gerada com a saída gold.
  • Teste automatizado: pipeline validado com caso em que a recuperação retorna os PRs `[1, 2, 4]` e alcança `Recall@K = 1.00` e `Exact Match = 1.00`.
  • Gancho de entrevista: desenho de pipeline de NLP aplicado a produto, retrieval explicável, heurística supervisionada por domínio e avaliação reproduzível.

Fake News Detection com PyTorch e GRU

Desenvolvimento de um notebook para classificação binária de notícias falsas versus confiáveis, estruturado como desafio de machine learning com foco em pipeline textual, rede neural e geração de arquivo de submissão.

  • Problema: identificar automaticamente notícias falsas a partir de título, conteúdo e tags, com avaliação orientada por F1-score.
  • Pré-processamento: tratamento de nulos, unificação de múltiplas colunas textuais em uma feature única e construção manual de vocabulário com até 10 mil tokens.
  • Arquitetura: embedding em PyTorch, camada recorrente `GRU`, saída sigmoide para classificação binária e treinamento com `BCELoss` e `Adam`.
  • Pipeline: split treino-validação, `DataLoader`, encoding com `PAD` e `UNK`, inferência em lote no conjunto de teste e geração de `submissions.csv`.
  • Stack: Python, pandas, PyTorch, scikit-learn, train_test_split, f1_score e notebook em Colab.
  • Gancho de entrevista: NLP clássico com deep learning, diferenças entre bag-of-words e embeddings, escolha de GRU para sequência textual e limitações de datasets pequenos.

  • Gancho de entrevista: como estruturar um servidor MCP, quando usar BM25 para documentação interna e como transformar conhecimento estático em interface consumível por agentes.

RAG NLP SQL com LangChain, OpenAI e SQLite

Desenvolvimento de uma aplicação em Python para responder perguntas em linguagem natural sobre uma base SQL, combinando recuperação de contexto semântico do schema com geração e execução de SQL por agent.

  • Problema: permitir consulta analítica em banco relacional sem exigir que o usuário escreva SQL manualmente.
  • Arquitetura: banco `SQLite` de exemplo, documentos semânticos sobre schema e regras de negócio, `BM25Retriever` para recuperar contexto e `create_sql_agent` do LangChain para gerar e executar consultas.
  • Stack: Python, LangChain, langchain-openai, langchain-community, SQLAlchemy, SQLite, Streamlit e BM25Retriever.
  • Diferencial técnico: separação clara entre dados estruturados e contexto de negócio, usando RAG não para documentos externos, mas para orientar melhor a geração de SQL.
  • Modo de uso: projeto com `modo demo` funcionando mesmo sem chave OpenAI e `modo completo` com geração dinâmica de SQL quando a API está configurada.
  • Segurança e adaptabilidade: o app não sobrescreve automaticamente bancos definidos pelo usuário e foi desenhado para troca simples da base e dos documentos de contexto.
  • Gancho de entrevista: como aplicar RAG fora do cenário clássico de PDFs, como reduzir ambiguidade na geração de SQL e como desenhar uma interface NL2SQL mais confiável.
